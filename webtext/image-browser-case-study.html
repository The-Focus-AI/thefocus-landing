Process

How it works

1. Extract from WARC archives

Used our custom warc_viewer tool to parse Internet Archive WARC files and extract 79,832 images from the FFFFound collection.

2. Generate embeddings

Each image becomes a 768-dimensional vector via Replicate's CLIP model. These vectors capture visual and semantic meaning.

3. Store in pgvector

Supabase PostgreSQL with pgvector extension. Multi-bucket design lets each collection have its own table.

4. Semantic search

Text queries get converted to embeddings in the same space. Cosine similarity finds visually relevant images.

Why CLIP?

CLIP creates embeddings in a shared latent space for both images and text. This means a text query like "sunset over ocean" and an actual sunset photo exist as nearby points in 768-dimensional space. Cosine similarity reveals semantic meaning.

Features

Production-ready design

Multi-bucket architecture

Each R2 bucket gets its own PostgreSQL table. Clean multi-tenancy ready for infinite collections.

Smart retry logic

Handles 429 rate limits and 5xx errors with exponential backoff. Automatic recovery for failed embeddings.

Concurrent processing

8 workers for uploads, 3 for embeddings. Optimized for Replicate API limits and throughput.

Fast vector search

pgvector handles 92K+ vectors with sub-100ms queries. Cosine distance operator for similarity.
